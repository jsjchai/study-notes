## Transformer架构

### 什么是Transformer

Transformer是一种基于自注意力机制的深度学习架构，由Vaswani等人于2017年提出，最初用于NLP任务。

### 核心组件

```
Transformer
├── Encoder（编码器）
│   ├── Multi-Head Self-Attention
│   ├── Feed Forward Network
│   └── Add & LayerNorm
│
└── Decoder（解码器）
    ├── Multi-Head Self-Attention
    ├── Cross-Attention
    ├── Feed Forward Network
    └── Add & LayerNorm
```

### LLM中的Transformer

现代大语言模型通常只使用Decoder部分（如GPT系列），采用单向注意力机制。

## 注意力机制

### Self-Attention（自注意力）

自注意力机制允许模型在处理序列时关注序列中的其他位置。

```
Attention(Q, K, V) = softmax(QK^T / √d_k) V
```

- **Q（Query）**：查询向量，表示当前位置
- **K（Key）**：键向量，用于匹配
- **V（Value）**：值向量，包含实际信息
- **d_k**：向量维度，用于缩放

### Multi-Head Attention（多头注意力）

多个注意力头并行工作，捕捉不同类型的关系：

```python
MultiHead(Q, K, V) = Concat(head_1, ..., head_h) W^O

head_i = Attention(QW_i^Q, KW_i^K, VW_i^V)
```

### 注意力类型

| 类型 | 描述 | 应用 |
|------|------|------|
| Self-Attention | 序列内部关注 | Transformer Encoder |
| Cross-Attention | 编码器-解码器关注 | Transformer Decoder |
| Causal Attention | 单向注意力 | GPT类模型 |

## 预训练与微调

### 预训练阶段

#### 目标
让模型学习语言的通用表示，掌握语法、语义、世界知识。

#### 训练任务

1. **Next Token Prediction（下一个token预测）**
   ```
   给定: "今天天气很好，我准备去"
   预测: "逛街"
   ```

2. **Masked Language Modeling（掩码语言模型）**
   ```
   给定: "今天[MASK]很好"
   预测: "天气"
   ```

#### 训练数据

- 互联网文本
- 代码库
- 书籍
- 百科

### 微全书调阶段

#### 指令微调（Instruction Tuning）

```
输入: "请总结以下文章：..."
输出: "文章摘要..."
```

#### 对齐微调（Alignment Tuning）

使用RLHF使模型输出更符合人类期望。

## RLHF人类反馈强化学习

### RLHF流程

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│  预训练模型  │───▶│  奖励模型   │───▶│  PPO训练   │
└─────────────┘    └─────────────┘    └─────────────┘
      ↑                                        │
      └────────────────────────────────────────┘
              （策略梯度更新）
```

### 步骤详解

#### 1. 收集人类反馈

```python
# 生成多个回复
responses = model.generate(prompts)

# 人类排序/打分
human_preferences = [
    (response_a, response_b, preference="A"),  # A更好
    (response_c, response_d, preference="D"),  # D更好
]
```

#### 2. 训练奖励模型（Reward Model）

```python
# 使用人类偏好数据训练
# 目标：预测人类偏好
reward_model = train_reward_model(prompts, responses, preferences)
```

#### 3. PPO强化学习

```python
# 使用PPO算法优化策略
policy = ppo_train(
    policy=pretrained_model,
    reward_model=reward_model,
    prompts=prompts
)
```

### RLHF的作用

- 提高指令遵循能力
- 减少有害输出
- 改善对话质量
- 使输出更自然

## 模型推理与部署

### 推理优化技术

#### 1. 量化（Quantization）

| 方法 | 精度损失 | 效果 |
|------|----------|------|
| FP16 | 极小 | 2x |
| INT8 | 小 | 4x |
| INT4 | 中等 | 8x |

#### 2. 蒸馏（Distillation）

```
教师模型(大) ──▶ 学生模型(小)
     │              │
     └─ 知识转移 ──┘
```

#### 3. 投机解码（Speculative Decoding）

```
小模型快速生成 ──▶ 大模型验证/修正
```

### 部署方式

| 方式 | 延迟 | 成本 | 适用场景 |
|------|------|------|----------|
| 云端API | 低 | 按量 | 生产环境 |
| 本地部署 | 中 | 硬件 | 隐私/离线 |
| 边缘设备 | 高 | 设备 | 移动端 |
