## 模型架构的差异

### Decoder-only vs Full Stack

```
Transformer架构选择：
┌─────────────────────────────────────────────────────────┐
│                    Transformer                          │
├─────────────────────────────────────────────────────────┤
│  Decoder-only (GPT系列)     │  Encoder-Decoder (T5)   │
│  ├─ 单向注意力              │  ├─ 双向注意力(Enc)    │
│  ├─ 因果语言建模            │  ├─ 单向注意力(Dec)    │
│  └─ 适合生成任务            │  └─ Seq2Seq任务        │
└─────────────────────────────────────────────────────────┘
```

### 注意力机制的进化

#### 标准Attention
```
Attention(Q,K,V) = softmax(QK^T / √d)V
```

#### 改进版本

| 机制 | 原理 | 代表模型 |
|------|------|----------|
| FlashAttention | IO-aware计算 | Llama |
| Grouped Query Attention | 共享K/V | Mistral |
| Multi-Query Attention | 极简K/V | PaLM |
| Sliding Window | 局部注意力 | Bloomberg |

### 位置编码

```
位置编码方式：
│
├── 绝对位置 (Sinusoidal)
│   └── 固定位置，无法外推
│
├── 旋转位置编码 (RoPE)
│   └── 旋转矩阵，可外推
│   └── 效果：Llama, GLM
│
└── 相对位置 (ALiBi)
    └── 距离偏置，外推能力强
    └── 效果: GPT-NeoX
```

## 推理策略的不同

### 采样方法

#### 确定性采样

```
Greedy Search
└── 始终选择最高概率token
    └── 优点：稳定
    └── 缺点：可能重复
```

#### 随机性采样

```
Temperature Sampling
└── T=1: 标准概率分布
└── T<1: 尖锐，确定性高
└── T>1: 平滑，随机性强

Top-K Sampling
└── 只从概率最高的K个token中采样

Top-P (Nucleus) Sampling
└── 累积概率达到P的最小token集合中采样
```

#### 高级策略

```
Beam Search
└── 保留多条候选路径

Contrastive Search
└── 平衡流畅度和多样性

Speculative Decoding
└── 小模型快速生成，大模型验证
```

### 推理优化

| 技术 | 原理 | 效果 |
|------|------|------|
| 量化 | 降低精度 | 2-8x加速 |
| 蒸馏 | 小模型学习大模型 | 压缩 |
| 剪枝 | 去除冗余参数 | 加速 |
| 缓存 | KV Cache | 减少重算 |

## 能力涌现的原理

### 什么是涌现

```
模型规模与能力关系：
│
│                              ┌──────────┐
│                         ┌────┤ 涌现能力 │────┌──────────┐
│                    ┌────┤         └────┘   │  推理    │
│               ┌────┤    └───────────────────┘   └──────────┘
│          ┌────┤    │                    ┌──────────┐
│     ┌────┤    │    │                    │  代码    │
│────┤    │    │    │                    └──────────┘
│ 基础 │    │    │    │
│能力  └────┴────┴────┴───────────────────────────────▶
     1B      10B      70B                 模型参数量
```

### 涌现能力举例

| 能力 | 涌现规模 | 解释 |
|------|----------|------|
| 思维链 | ~60B | 复杂推理需要足够参数 |
| 指令遵循 | ~8B | 足够容量理解指令 |
| 代码生成 | ~20B | 编程语法需要足够表达 |
| 多语言 | ~30B | 跨语言对齐需要规模 |

### 涌现的原理假说

1. **参数临界点假说**
   - 达到一定规模后，神经网络形成新结构

2. **任务分解能力**
   - 大模型可隐式分解复杂任务

3. **知识整合**
   - 互联网规模数据中的知识被整合

## 总结

### 核心观点

1. **架构决定基础**
   - 注意力变体、位置编码影响模型特性

2. **数据塑造能力**
   - 训练数据配比决定能力倾向

3. **规模带来涌现**
   - 参数规模突破临界点后出现新能力

4. **推理策略影响体验**
   - 采样方法决定输出风格

### 理解多样性的意义

- 选择模型时看原理而非表面
- 根据任务需求选择架构
- 理解能力边界，合理期望
