## 📝 笔记添加规范

### 1. 新笔记创建流程
```bash
# 1. 进入对应年份目录
cd 2026

# 2. 使用模板创建新笔记
cp ../templates/技术教程模板.md 新主题笔记.md

# 3. 编辑笔记内容
# 4. 更新年度README索引
# 5. 提交并推送更改
git add .
git commit -m "添加2026年笔记：[主题]"
git push origin master
```

### 2. 笔记质量标准
- ✅ **内容完整**: 主题明确，结构清晰
- ✅ **代码规范**: 代码示例完整，可运行
- ✅ **格式统一**: 使用Markdown标准格式
- ✅ **链接有效**: 所有引用链接可访问
- ✅ **分类正确**: 按技术领域正确分类

### 3. 模板使用指南
- **技术教程模板**: 用于技术操作指南、安装配置教程
- **学习总结模板**: 用于课程学习、书籍阅读总结

## 🔄 同步管理

### 1. 日常同步流程
```bash
# 拉取远程更新
git pull origin master

# 添加本地更改
git add .

# 提交更改
git commit -m "更新: [简要描述]"

# 推送更改
git push origin master
```

### 2. 冲突解决策略
1. **优先保留远程更改**（如果冲突较小）
2. **手动合并冲突**（如果内容重要）
3. **创建备份分支**（如果冲突复杂）
