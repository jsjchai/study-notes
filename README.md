## 📝 笔记规范

### 文件结构
- 按年份组织，每年一个目录
- 按技术领域分类，每个领域一个子目录
- 使用Markdown格式，UTF-8编码

### 内容标准
1. **技术教程**: 步骤清晰，代码完整
2. **读书笔记**: 要点总结，个人思考
3. **问题思考**: 问题描述，解决方案
4. **项目实践**: 项目背景，实现过程

### 模板系统
提供标准化模板，确保笔记质量：
- `templates/技术教程模板.md` - 技术文档模板
- `templates/学习总结模板.md` - 学习记录模板

## 🚀 使用方法

### 添加新笔记
```bash
# 进入对应年份目录
cd 2026

# 创建新笔记
touch 新主题笔记.md

# 使用模板创建
cp ../templates/技术教程模板.md 新主题笔记.md
```

### 更新索引
每次添加重要笔记后，更新对应年份的README.md文件。

### 版本控制
使用Git进行版本管理，定期提交更新。
